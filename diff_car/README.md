# diff_car - 差速驱动机器人导航与控制

## 项目简介

`diff_car` 是一个轻量级的差速驱动机器人 ROS2 功能包。项目提供了简洁高效的导航解决方案，支持 A* 自动寻路、预设航点巡迹和键盘手动控制等多种模式。实现了基于 ROS2 的 A* 全局规划器，包含障碍物膨胀、8 邻域启发式搜索、PI 航向跟踪与积分抗饱和，在 Gazebo 差速底盘仿真中验证了绕障与轨迹跟踪能力。后续还会补充 Nav2 方案的激光雷达导航功能。

## 技术亮点

### 1. 完全独立的导航架构
- **PI 控制器**：带积分限幅、积分分离和输出限幅的高级控制器，确保稳定控制
- **轻量化设计**：仅依赖 `rclpy`、`geometry_msgs`、`nav_msgs`
- **开箱即用**：无需配置复杂的导航参数和地图

### 2. 多种导航模式

#### A* 自动寻路
- **完整 A* 算法实现**：支持 8 方向搜索（对角线移动）
- **自建栅格地图**：80×80 网格，0.2m 分辨率
- **自动避障**：基于机器人半径的安全距离计算
- **动态路径规划**：从起点到终点自动计算最优路径

#### 预设航点巡迹
- **固定路线导航**：预定义 12 个航点绕过障碍物
- **平滑轨迹**：自动计算转向角度
- **高精度到达**：位置容差 0.15m

#### 键盘手动控制
- **实时交互**：支持 WASD/IJKL 双键位
- **S 曲线插值**：五次多项式平滑加速，无速度阶跃
- **加速时间**：0.3 秒（可调）

### 3. 高级 PI 控制器

#### 带积分限幅、积分分离和输出限幅的 PI 控制
```python
# 核心参数
kp = 2.5                    # 比例增益
ki = 0.3                    # 积分增益
integral_max = 1.0          # 积分上限（防饱和）
integral_min = -1.0         # 积分下限（防饱和）
integral_separation_threshold = 0.2  # 积分分离阈值
```

**技术特性：**
- ✅ **积分分离**：大误差时（>0.2 rad）禁用积分，加快响应速度
- ✅ **积分限幅**：限制积分项在 [-1.0, 1.0]，防止积分饱和
- ✅ **输出限幅**：PI 控制器输出限制在 [-angular_speed, angular_speed]
- ✅ **智能重置**：每个航点开始时清零积分，避免累积误差
- ✅ **稳态误差消除**：小误差时启用积分，精确到达目标

**完整控制策略：**
```
┌─────────────────────────────────────────────────────────┐
│ 1. 路径规划层                                            │
│    输入：起点、终点                                      │
│    输出：路径点列表 [(x1,y1), (x2,y2), ...]             │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ 2. 航点跟踪层                                            │
│    判断：distance_to_target > 0.15m ?                   │
│    ├─ 是 → 继续跟踪当前航点                              │
│    └─ 否 → 切换到下一个航点                              │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ 3. 角度计算层                                            │
│    target_yaw = atan2(target_y - current_y,             │
│                       target_x - current_x)             │
│    yaw_error = target_yaw - current_yaw                 │
│    含义：当前朝向与目标方向的偏差                         │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ 4. 决策层（分支控制）                                    │
│    ┌───────────────────────────────────────────────┐   │
│    │ |yaw_error| > 0.5 rad (28.6°)                 │   │
│    │   → 原地旋转（线速度=0，角速度=±0.3 rad/s）    │   │
│    │   → 目的：快速对准目标方向                      │   │
│    │                                               │   │
│    │ |yaw_error| ≤ 0.5 rad                         │   │
│    │   → 前进 + 转向（线速度=0.15，角速度=PI控制）  │   │
│    │   → 目的：边前进边调整方向                      │   │
│    └───────────────────────────────────────────────┘   │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ 5. PI 控制层（仅在小误差时生效）                          │
│    输入：yaw_error（角度误差）                            │
│    输出：cmd.angular.z（角速度）                          │
│                                                         │
│    |yaw_error| > 0.2 rad → 仅 P 控制（积分分离）         │
│    |yaw_error| ≤ 0.2 rad → P + I 控制                   │
│    最后：输出限幅到 [-0.3, 0.3] rad/s                   │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ 6. 执行层（差速驱动）                                    │
│    发布：Twist() 到 /diff_cont/cmd_vel_unstamped        │
│                                                         │
│    cmd.linear.x  = 0.15 m/s  (前进速度)                 │
│    cmd.angular.z = PI输出  (转向速度)                    │
│                                                         │
│    差速控制器内部计算左右轮速度：                          │
│    左轮 = linear.x - angular.z × wheel_separation/2     │
│    右轮 = linear.x + angular.z × wheel_separation/2     │
└─────────────────────────────────────────────────────────┘
```

### 4. S 曲线插值键盘控制

#### 五次多项式平滑加速
```python
# S 曲线公式
s(t) = 10t³ - 15t⁴ + 6t⁵  (t ∈ [0,1])

# 核心参数
accel_time = 0.3           # 加速时间（秒）
max_linear_speed = 0.15    # 最大线速度
max_angular_speed = 0.3    # 最大角速度
```

**技术特性：**
- ✅ **五次多项式**：起点和终点加速度为 0，无突变
- ✅ **平滑加速**：速度连续变化，消除阶跃感
- ✅ **参数可调**：通过 `accel_time` 调整响应速度
- ✅ **与导航一致**：速度参数与自动导航模式相同

| 潜在问题      | 说明                                        |
| --------- | ----------------------------------------- |
| ⚠️ 插值时间固定 | 0.3s 对小速度变化（如 0.05→0）也走完整 S 曲线，可能显得"迟钝"   |
| ⚠️ 无紧急制动  | 按 `k` 停止也是 0.3s 减速，不是立即归零                 |
| ⚠️ 键盘无并发  | 不支持同时按 `w` 和 `a`（对角前进），因为 `read(1)` 只读单字符 |
| ⚠️ 循环频率不稳 | `spin_once` + `select` 的组合实际周期可能有抖动       |

- **动态插值时间方案**：根据速度差 ∣Δv∣，自适应调整 accel_time
- **急停方案**：单独加一个"急停"逻辑，绕过 S 曲线直接归零

### 5. 精确的位姿估计
- **双里程计订阅**：同时监听 `/odom` 和 `/diff_cont/odom`
- **四元数转欧拉角**：精确计算航向角
- **角度归一化**：确保角度在 [-π, π] 范围内

### 6. 模块化设计

**四种独立启动模式：**
```bash

ros2 launch diff_car demo_astar.launch.py

# 模式2: 预设航点巡迹（带障碍物场景）

# 模式3: 键盘手动控制（空场景）

# 模式4: 纯仿真环境（无导航）

```

## 使用方法

### 1. A* 自动寻路
```bash
ros2 launch diff_car demo_astar.launch.py
```
**场景说明：**
- 起点：(0, 0)
- 终点：(6, 0)
- 障碍物：(3, 0)，半径 0.3m
- 行为：机器人自动规划路径，绕过障碍物到达目标点

### 2. 预设航点巡迹
```bash
ros2 launch diff_car demo_simple.launch.py
```
**场景说明：**
- 起点：(0, 0)
- 终点：(6, 0)
- 障碍物：(3, 0)，半径 0.3m
- 行为：机器人沿预设的 12 个航点巡迹，绕过障碍物到达终点

### 3. 键盘控制
```bash
ros2 launch diff_car demo_keyboard.launch.py
```
**按键说明：**
- `i` / `w` - 前进
- `,` / `s` - 后退
- `j` / `a` - 左转
- `l` / `d` - 右转
- `k` - 停止
- `q` - 退出

**控制特性：**
- 采用五次多项式 S 曲线插值，速度平滑变化
- 加速时间 0.3 秒，无速度阶跃
- 速度参数与导航模式一致（0.15 m/s, 0.3 rad/s）

### 4. 纯仿真环境
```bash
ros2 launch diff_car launch_sim.launch.py
```
**用途：** 仅启动 Gazebo 仿真环境，不启动任何导航节点，用于手动测试或二次开发

## 常见问题

### Q1: 小车到达航点时不够精确怎么办？
A: 可以调整 `position_tolerance` 参数（默认 0.15m），减小该值可以提高精度，但可能导致小车在航点附近振荡。

### Q2: 小车转向不够快怎么办？
A: 可以增大 `kp` 参数（默认 2.5），但过大可能导致振荡。建议每次增加 0.2 进行测试。

### Q3: 小车在航点附近来回摆动怎么办？
A: 可以减小 `ki` 参数（默认 0.3）或增大 `integral_separation_threshold`（默认 0.2），减少积分作用。

### Q4: 如何修改起点和终点？
A: 在 `astar_navigator.py` 中修改 `self.start` 和 `self.goal` 变量，或在 `demo_simple_navigator.py` 中修改 `self.waypoints` 列表。
